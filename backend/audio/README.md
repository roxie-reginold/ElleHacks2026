# Generated Audio Files

This directory stores audio files generated by ElevenLabs text-to-speech.

## Directory Structure

```
backend/audio/
├── .gitkeep                    # Preserves directory in git
├── README.md                   # This file
└── elevenlabs_*.mp3           # Generated audio files (gitignored)
```

## File Naming Convention

Audio files are automatically named with the following format:
```
elevenlabs_<timestamp>.mp3
```

Example: `elevenlabs_1769885972184.mp3`

## Git Ignore Rules

Audio files (`*.mp3`, `*.wav`, `*.ogg`) are excluded from git via `.gitignore` to prevent bloating the repository. Only the directory structure (`.gitkeep` and `README.md`) is tracked.

## Access

Audio files can be accessed via:
- **API endpoint**: `GET /api/tts/audio/:filename`
- **Direct path**: `backend/audio/elevenlabs_<timestamp>.mp3`

Example:
```bash
# Via API
curl http://localhost:3001/api/tts/audio/elevenlabs_1769885972184.mp3 -o output.mp3

# Direct file access
open backend/audio/elevenlabs_1769885972184.mp3
```

## Cleanup

Audio files are persistent. To clean up old files:

```bash
# Remove all generated audio files
rm backend/audio/elevenlabs_*.mp3

# Remove files older than 7 days
find backend/audio -name "elevenlabs_*.mp3" -mtime +7 -delete
```

## Production Considerations

For production deployments:
1. **Storage**: Consider using cloud storage (S3, Google Cloud Storage) for audio files
2. **Cleanup**: Implement automatic cleanup of old files
3. **CDN**: Serve audio files through a CDN for better performance
4. **Caching**: Cache frequently used calming prompts to reduce API calls

## Security

Only files starting with `elevenlabs_` can be accessed via the API endpoint for security reasons.
